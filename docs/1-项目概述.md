# ShortPlay - AI 短剧生成平台

## 1. 项目概述

### 1.1 项目简介

**ShortPlay** 是一个面向短剧创作的全流程AI生成平台，覆盖从文本输入到视频生成的整体流程。

**核心功能流程：**
```
文本输入 → 信息提取 → 角色/场景资产生成 → 分镜管理 → 视频生成 → 视频编辑
```

### 1.2 技术栈

| 层级 | 技术选型 |
|------|----------|
| 后端框架 | FastAPI (Python 3.11+) |
| 前端框架 | React 18 + TypeScript |
| UI框架 | TailwindCSS + shadcn/ui |
| 数据库 | PostgreSQL 15 |
| 缓存 | Redis 7 |
| 消息队列 | RabbitMQ 4.x |
| 搜索引擎 | Elasticsearch 8.11 (IK+pinyin) |
| 向量数据库 | Qdrant |
| 对象存储 | MinIO |
| 容器化 | Docker + Docker Compose |

## 2. 功能模块说明

### 2.1 项目管理 (Project)

| 功能 | 描述 |
|------|------|
| 创建项目 | 创建新的短剧项目 |
| 编辑项目 | 修改项目名称、描述、封面 |
| 删除项目 | 软删除项目及其关联数据 |
| 项目列表 | 分页、关键词搜索、状态筛选 |
| 项目搜索补全 | Elasticsearch suggest |

### 2.2 章节管理 (Episode)

| 功能 | 描述 |
|------|------|
| 创建章节 | 在项目下创建新章节 |
| 编辑章节 | 修改章节名称、内容 |
| 删除章节 | 软删除章节 |
| 步骤状态维护 | 跟踪当前处理步骤 |
| 步骤跳转 | 根据进度约束允许跳转 |

### 2.3 信息提取 (Extraction)

| 功能 | 描述 |
|------|------|
| 提取角色 | 从章节文本提取角色信息 |
| 提取场景 | 从章节文本提取场景信息 |
| 重新提取 | 支持重新提取并覆盖 |
| 手动选择模型 | 可选择文本模型 |

### 2.4 资产管理 (Asset)

| 功能 | 描述 |
|------|------|
| 角色管理 | 增删改查、生成头像、重生 |
| 场景管理 | 增删改查、生成缩略图、重生 |
| 图片上传 | 支持自定义图片替换 |
| 资产搜索 | 项目内Elasticsearch搜索 |

### 2.5 分镜管理 (Storyboard)

| 功能 | 描述 |
|------|------|
| 生成分镜 | 按章节生成镜头脚本 |
| 向量检索增强 | 可选使用Qdrant向量检索 |
| 镜头配置 | 镜头类型、时长、描述 |
| 角色关联 | 关联最多3个角色 |
| 场景关联 | 必须且仅关联1个场景 |

### 2.6 视频生成 (Video)

| 功能 | 描述 |
|------|------|
| 首帧提示词 | 生成可编辑的首帧描述 |
| 首帧图片生成 | 生成视频首帧 |
| 分镜视频生成 | 生成分镜视频任务 |
| 任务状态推送 | WebSocket实时推送 |
| 错误展示 | 显示后端错误信息 |

### 2.7 模型管理 (Model)

| 功能 | 描述 |
|------|------|
| 厂商管理 | 支持多厂商配置 |
| 模型实例 | TEXT/IMAGE/VIDEO/AUDIO类型 |
| 默认模型 | 按类型设置默认实例 |
| 参数映射 | 自定义模型参数 |

### 2.8 提示词管理 (Prompt)

| 功能 | 描述 |
|------|------|
| 模板管理 | 多场景提示词模板 |
| 默认提示词 | 设置场景默认模板 |
| 模板市场 | 预设常用模板 |

## 3. 数据流

### 3.1 创作流程

```
1. 用户创建项目 + 章节
2. 提交章节文本 → 信息提取
   ├── 调用LLM提取角色 + 场景
   └── 保存到数据库 + ES索引
3. 生成角色/场景图片
   ├── 选择风格模板
   └── 调用图像生成API
4. 生成分镜
   ├── 关联角色 + 场景资产
   └── 调用LLM生成镜头脚本
5. 生成视频
   ├── 生成首帧提示词 + 图片
   ├── 生成分镜视频
   └── WebSocket推送状态
6. 视频编辑 (规划中)
```

### 3.2 消息队列流

```
用户请求 → API → RabbitMQ → Worker处理 → 回调通知
                              ↓
                        第三方API轮询
                              ↓
                        状态更新 → WebSocket推送
```
